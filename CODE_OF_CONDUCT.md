# Code of Conduct

goDrive should be a practical and respectful project for users and contributors.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but keep technical discussions concrete.
- Critique code, designs, and tradeoffs rather than people.
- Keep issues and pull requests focused on the project.
- Disclose security issues privately through `SECURITY.md`.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory or sexualized language or imagery.
- Publishing private information without explicit permission.
- Repeated off-topic disruption after maintainers ask for it to stop.
- Publicly posting exploit details for unresolved vulnerabilities.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or restrict participation when behavior harms the project or its contributors.

If you need to report a conduct issue, contact the maintainers through the repository owner profile or another private channel listed by the project.
